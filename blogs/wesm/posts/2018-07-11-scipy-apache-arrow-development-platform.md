## Summary

In this talk, I explain the motivation behind Apache Arrow and how it's evolving into a development platform for data processing systems. I start by discussing why open standards matter: they simplify architecture, reduce ecosystem fragmentation, enable code reuse, and make systems more interoperable. I note that CSV is not actually an open standard despite what people might think, and SQL has inconsistent implementations across databases.

I cover why columnar data formats matter for analytics. Column-oriented tables enable better CPU cache utilization because analytical queries typically scan all values in a column or operate on just a few columns of wide tables. This can mean 10-100x performance improvements from better memory access patterns alone, even comparing linear-time algorithms. Columnar layouts also enable SIMD vectorization and specialized compression techniques.

I explain the problem Arrow solves: pandas, R, Julia, and every SQL database has its own internal representation of tables. These aren't interoperable—you can't run an R algorithm on a pandas DataFrame. If you have N systems and want them all to talk to each other, you need N-squared converters. Arrow provides a common format that reduces this to N converters. The goal is zero-copy data sharing between systems.

I describe what we've built: a columnar memory format specification that's language-independent, with implementations in 8+ languages. The project started in early 2016 with about 25 people from various big data projects, done under the Apache Software Foundation because many contributors work for competing companies. By 2018, we had nearly 200 contributors and over 100,000 monthly PyPI installs.

I showcase some capabilities: Dremio built an LLVM compiler that generates native Arrow functions in C++, called from Java, with data passed through JNI without copying. The Plasma object store (from Ray, donated to Arrow) enables zero-copy shared memory for tensors. NVIDIA and partners like Anaconda, MapD, and Graphistry are using Arrow to build GPU DataFrames with zero-copy workflows.

I discuss our technical work on Parquet support (with help from Blue Yonder), and announce Ursa Labs, the organization I created to fund full-time Arrow developers in partnership with RStudio and with support from Two Sigma. The goal is defragmenting the ecosystem so we can build reusable libraries and portable systems together.
