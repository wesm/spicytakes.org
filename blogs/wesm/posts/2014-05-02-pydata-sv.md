## Summary

This tutorial covers pandas basics. My O'Reilly book features a golden-tailed tree shrew ("I thought maybe if O'Reilly had a sense of humor, they would have put a giant panda on the cover").

## Core Concepts

pandas is "a rich data manipulation tool built on top of NumPy" providing SQL-like and R-like operations that were "historically very difficult to do in Python." Two data structures: **Series** (one-dimensional labeled array) and **DataFrame** (two-dimensional tabular structure).

**Automatic data alignment** through labeled indexes: when adding two Series with different indexes, pandas performs a join, inserting NaN where labels don't match. "You can switch to Python and forget about ismember," referencing MATLAB's alignment function.

## GroupBy

The **split-apply-combine** paradigm:
- Split data into groups based on criteria
- Apply functions to each group
- Combine results

Specialized `aggregate` and `transform` methods exist because "if you know that you're aggregating, you know that you're transforming," allowing performance optimizations.

## Baby Names Analysis

Using 258,000 records spanning 130 years: John dominated as the most popular boys' name in 1880 (8.15%), while by 2008 Jacob held the top spot with only about 1%—demonstrating increasing name diversity.

**Label-based indexing** through `.ix` supports both integer positions and string labels. Performance note: `get_value()` runs about 1.25 microseconds faster than bracket indexing—"You can tell I've spent far too much thinking about this."

## Missing Values

`isNull` and `notNull` work across data types unlike NumPy's `isNaN` (limited to floating-point). Automatic alignment produces the union of indexes, filling gaps with NaN.

## Limitations

"The data that we're working with, it has to fit into memory." Interest in extending to memory maps, large data stores, and distributed computation.

## Anti-DIY Philosophy

Against expecting users to write "six or eight lines of matplotlib" for common tasks. Instead, `.plot()` leverages labeled data structure for visualizations with minimal code.
