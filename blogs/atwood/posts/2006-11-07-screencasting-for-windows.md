If a picture is worth a thousand words, **is a single screencast equal to a thousand word blog post?**


There’s a lot to be said for lightweight, embedded screencasts. I’m particularly fond of animated GIF screencasts for small demonstrations. You can see examples in these posts: [one](https://blog.codinghorror.com/search-if-it-isnt-incremental-its-excremental/), [two](https://blog.codinghorror.com/code-snippets-in-vsnet-2005/), [three](https://blog.codinghorror.com/invisible-formatting-tags-are-evil/).


Here’s a compendium of all the screencasting tools I could find, but it’s far from comprehensive:

- [Camtasia Studio](http://www.techsmith.com/products/studio/default.asp) ($300)
- [BB FlashBack](http://www.bbsoftware.co.uk/BBFlashBack.aspx) ($40-300)
- [Adobe Captivate 2](http://www.adobe.com/products/captivate/) ($500)
- [Super Screen Recorder](http://www.free-screen-capture.com/screen-recorder/) ($40)
- [oRipa Screen Recorder](https://web.archive.org/web/20061117043232/http://www.ejoystudio.com/oripa-screen-recorder/index.htm) (free)
- [CamStudio Desktop Screen Recorder](http://sourceforge.net/projects/camstudio/) (free)
- [Animated GIF plugin for Cropper](http://weblogs.asp.net/jgalloway/archive/2006/11/07/Animated-GIF-Plugin-for-Cropper-_2B00_-some-.NET-Animated-GIF-code.aspx) (free)
- [GifgIfgiF](http://www.peda.com/ggg/) ($28)
- [Wink](http://www.debugmode.com/wink/) (free)
- [DemoCharge](http://www.yessoftware.com/products/product_detail.php?product_id=19) ($50-$200)


You’re also probably wondering **which of these tools I recommend**. What do I look like, Scott Hanselman? I’ve only used three of these, not all of them. Check out [Donation Coder’s screencasting roundup](https://web.archive.org/web/20061117134014/http://www.donationcoder.com/Reviews/Archive/ScreenCasting/) for a blow by blow comparison.


Personally, I use **GifgIfgiF for quick and dirty animated GIF screencasts**, and **Camtasia studio** for more advanced screencasts where I may need to do editing or render to output formats such as Flash, Quicktime, or Windows Media. One of the biggest challenges in screencasting is choosing an appropriate codec. Video codecs are optimized for movie content, *definitely* not for GUIs. Screencast results can be quite poor if you choose a typical movie codec. Did you know Windows Media Player has an outstanding [screen codec](https://web.archive.org/web/20070222182534/http://www.microsoft.com/technet/prodtechnol/windowsmedia/evaluate/scrcodec.mspx) that’s [optimized for GUIs](https://web.archive.org/web/20061205144648/http://www.microsoft.com/windows/windowsmedia/forpros/codecs/video.aspx#WindowsMediaVideo9Screen)?


I recommend picking up one of the free screencasting tools to start with. Add it to your toolkit. Get comfortable with how it works so you can whip out a screen capture session any time the situation calls for it. However, if you find yourself doing a lot of screencasting, I’d definitely invest in one of the more expensive tools that offers more editing options and more choices in output rendering.
