I’ve increasingly come to believe that the debate between C# and VB.NET is a [red herring](http://en.wikipedia.org/wiki/Red_herring). Choosing between C# and VB.NET isn’t a *meaningful* choice. **It’s like choosing between .NET Pepsi and .NET Coke**.


![](https://blog.codinghorror.com/content/images/2025/05/image-202.png)


Either way, *you’re getting a cola flavored carbonated beverage*.


If C# is Coke and VB.NET is Pepsi, then Python is [Yoohoo](http://www.drinkyoo-hoo.com/about/history.html). It’s more than mild differences in syntax. It’s a completely [dynamic language](http://en.wikipedia.org/wiki/Dynamic_language) with [Duck typing](http://en.wikipedia.org/wiki/Duck_typing) where indentation is used in place of squigglies or End statements.


Wasn’t this the promise of the .NET runtime – that we’d have **real, meaningful choice of language?** That’s why I’m so excited about [IronPython](http://www.ironpython.com/). IronPython has been a fine proof of concept, but it had no IDE integration. Until now.


As [documented](https://web.archive.org/web/20060220153818/http://blogs.msdn.com/aaronmar/archive/2006/02/16/533273.aspx) in Aaron Marten’s blog, the latest CTP of the [Visual Studio SDK](https://web.archive.org/web/20060328054216/https://affiliate.vsipmembers.com/login.aspx?err=cannot_find_user) finally **makes Python a first-class citizen in the Visual Studio 2005 IDE:**


![](https://blog.codinghorror.com/content/images/2025/05/image-203.png)


And here’s the requisite [Hello World](https://blog.codinghorror.com/the-real-cost-of-hello-world/) example:


![](https://blog.codinghorror.com/content/images/2025/05/image-204.png)


I can’t see any obvious way to build a web or winforms app in Python yet. But it’s coming.
