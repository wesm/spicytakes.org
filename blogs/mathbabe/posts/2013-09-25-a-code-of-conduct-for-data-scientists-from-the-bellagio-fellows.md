The 2013 PopTech & Rockefeller Foundation [Bellagio Fellows](http://www.poptech.org/bellagiofellows) – [Kate Crawford](http://poptech.org/people/kate_crawford), [Patrick Meier](http://poptech.org/people/patrick_meier), [Claudia Perlich](http://poptech.org/people/claudia_perlich), [Amy Luers](http://poptech.org/people/amy_luers), [Gustavo Faleiros](http://poptech.org/people/gustavo_figueiredo_faleiros) and [Jer Thorp](http://poptech.org/people/jer_thorp) – yesterday published [“Seven Principles for Big Data and Resilience Projects”](http://irevolution.net/2013/09/23/principles-for-big-data-and-resilience/) on [Patrick Meier’s blog iRevolution](http://irevolution.net/bio/).


Although they claim that these principles are meant for “best practices for resilience building projects that leverage Big Data and Advanced Computing,” I think they’re more general than that (although I’m not sure exactly what a resilience building project is) I and I really like them. They are looking for public comments too. Go to [the post](http://irevolution.net/2013/09/23/principles-for-big-data-and-resilience/) for the full description of each, but here is a summary:


**1. Open Source Data Tools**


Wherever possible, data analytics and manipulation tools should be open source, architecture independent and broadly prevalent (R, python, etc.).


**2. Transparent Data Infrastructure**


Infrastructure for data collection and storage should operate based on transparent standards to maximize the number of users that can interact with the infrastructure.


**3. Develop and Maintain Local Skills**


Make “Data Literacy” more widespread. Leverage local data labor and build on existing skills.


**4. Local Data Ownership**


Use Creative Commons and licenses that state that data is not to be used for commercial purposes.


**5. Ethical Data Sharing**


Adopt existing data sharing protocols [like the ICRC’s](http://irevolution.net/2013/04/11/data-protection-for-crisis-mapping/) (2013). Permission for sharing is essential. How the data will be used should be clearly articulated. An opt in approach should be the preference wherever possible, and the ability for individuals to remove themselves from a data set after it has been collected must always be an option.


**6. Right Not To Be Sensed**


Local communities have a right not to be sensed. Large scale city sensing projects must have a clear framework for how people are able to be involved or choose not to participate.


**7. Learning from Mistakes**


Big Data and Resilience projects need to be open to face, report, and discuss failures.
