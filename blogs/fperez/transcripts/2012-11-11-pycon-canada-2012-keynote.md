How many people here have heard of Cython or have used Cython? Good chunk of you. Cython is a very interesting tool. It allows us to take code which is in Python syntax and, adding a minimal amount of type information, generates C that is statically typed and can therefore be compiled to be faster. But Cython understands the entire Python API, so whatever code is pure Python you can keep in Python, and you can add type declarations only as you need them.

Let's imagine that this is taken from their tutorial -- this is a typical case of a function that is very simple to write. It's a numerical integrator that takes a function and integrates it between two points. This type of code that does a loop and a bunch of simple arithmetic is exactly the kind of code for which Python is catastrophically slow, even though it's very simple code. But this is the type of code for which Cython is perfect.

You grab that function and say run it in Cython. You put `%%cython` at the top and you simply copy and paste your code and add a few type declarations: X is always going to be a double, this function is always going to return a double, A and B are doubles, N is an int, I is an int, S and DX are doubles. That's it. You only added type information and a little bit of exception control. That's all you had to do.

Then if you run both versions, you'll notice that the Python version needs about 70 microseconds to execute one iteration. The Cython version needs under one microsecond. We're talking about close to a factor of 100 speedup with a minimal amount of effort, right there interactively in the browser. What we do is, if this code changes -- because these are extension modules and Python doesn't know how to reload an extension module -- under the hood, the actual module that gets imported is an MD5 hash of the source, so that if the source code changes, you just reload another module and you're guaranteed to have a clean object in memory. You don't have to restart your interpreter.

This has been our motivation. The way it's built is basically on top of two pillars. **A:** Yes. With the JavaScript integration that we're starting to have, our focus isn't so much the immediate visualization change by change.
