Daniel Eran Dilger at Roughly Drafted published a piece over the weekend [claiming](http://www.roughlydrafted.com/RD/RDM.Tech.Q3.07/DD5FFFA5-2CDA-46C7-ADE9-E4AD4BF01602.html) that “the iPod is running Mac OS X — with a Mach kernel and a Unix userland — and has been for years”. The story was [picked up by Slashdot](http://apple.slashdot.org/article.pl?sid=07/07/15/176216&threshold=-1) earlier today, originally with the title “iPods Run OS X and Always Have”.


Dilger’s evidence for this claim is based on [this April 2004 article in MacTech](http://www.mactech.com/articles/mactech/Vol.20/20.04/SnowDay/index.html) about a supposed method of developing QuickTime-based applications for iPods, using a programming language called “SNOJOB”, derived from SNOBOL. This MacTech article was a [*joke*](http://www.ilounge.com/index.php/news/comments/developing-quicktime-apps-for-ipod/), an unfunny April Fool’s [hoax](http://www.indopedia.org/Talk:IPod.html). Current iPods do not run OS X, or anything even vaguely resembling it. There is no programming language called SNOJOB. There is no secret “laf” option for the `ls` command-line tool that will reveal a “hidden” mach kernel on iPod file systems.1


---

1. There are ‘-l’, ‘-a’, and ‘-f’ options for `ls`, but using them won’t reveal anything that looks like a Darwin-ish OS hidden on an iPod. ↩︎
