I'm incredibly proud to present the first public release of
[Active Record](http://activerecord.rubyonrails.org/)
— an implementation of the
[pattern](http://www.martinfowler.com/eaaCatalog/activeRecord.html)
by the same name for object-relational mapping. I've entitled this release Three-Quarters to reflect the fact that Active Record has been in production use on
[Basecamp](http://www.basecamphq.com)
for more than three months and used during the many months of development before that. Additionally, it's been used by around five beta testers in a number of different projects. So even though I'm not ready to declare it 1.0 out of the gates, it is indeed a solid release.

It also marks the first step towards releasing the web-application framework
[Rails](http://www.rubyonrails.org)
, which I've been hyping and talking about for some time now. Active Record is the model part of Rails and contains around two-thirds of the code in the framework. So if you want to get ready for making Rails applications, it's a good idea to familiarize yourself with Active Record.
